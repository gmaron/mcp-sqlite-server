# Spec: Medium Export ETL Extractor (RAG Data Pipeline - Paso 1)

## 1. Contexto y Objetivo
Se requiere construir un script en Python (`medium_extractor.py`) que actúe como la primera fase de un pipeline de datos (ETL) para una arquitectura RAG (Retrieval-Augmented Generation). 
El objetivo es procesar un directorio local que contiene el export histórico de artículos de Medium en formato HTML, extraer el contenido semántico de valor, limpiarlo de etiquetas de maquetado y generar un archivo estructurado listo para la fase de vectorización (Embeddings).

## 2. Entradas (Input)
* **Directorio de origen:** El script debe leer una carpeta local (por defecto `./medium_export/posts/`).
* **Formato de origen:** Múltiples archivos con extensión `.html`.
* **Estructura típica de Medium:** Los archivos contienen mucho "ruido" (clases CSS generadas, metadatos, menús). El contenido real suele estar en etiquetas de encabezado (`<h1>`, `<h2>`, `<h3>`) y de párrafo (`<p>`, `<blockquote>`, `<ul>`, `<ol>`, `<li>`).

## 3. Procesamiento y Reglas de Negocio
El script debe iterar sobre cada archivo `.html` en el directorio de entrada y realizar las siguientes operaciones utilizando la librería `BeautifulSoup4`:

1. **Extracción del Título:** Buscar la etiqueta `<title>` o el primer `<h1>`. Limpiar el sufijo " - Medium" si estuviera presente.
2. **Extracción del Cuerpo (Body):** * Extraer únicamente el texto contenido dentro de las etiquetas de contenido semántico (`<p>`, `<h1>`, `<h2>`, `<h3>`, `<blockquote>`, `<li>`).
   * Ignorar explícitamente etiquetas `<script>`, `<style>`, `<nav>`, `<footer>` y cualquier metadato no visible.
3. **Limpieza y Sanitización:**
   * Eliminar todas las etiquetas HTML residuales (obtener texto plano).
   * Eliminar espacios en blanco redundantes, tabulaciones y saltos de línea múltiples (reemplazar por un solo espacio o un salto de línea limpio).
   * Manejar correctamente la codificación (UTF-8) para no romper tildes ni caracteres especiales en español.

## 4. Salidas (Output)
* **Destino:** Un único archivo local llamado `corpus_limpio.json`.
* **Estructura de Datos:** Una lista de diccionarios en formato JSON. Cada diccionario representará un artículo y deberá tener estrictamente esta estructura:
  ```json
  [
    {
      "id": "nombre-del-archivo-original",
      "titulo": "Título limpio del artículo",
      "contenido": "Texto plano sanitizado completo..."
    }
  ]

## 5. Requerimientos No Funcionales y Calidad de Código
Modularidad: El código debe estar dividido en funciones con responsabilidades únicas (ej. read_files(), parse_html(), save_json()).

Typing: Utilizar Type Hints de Python en todas las funciones.

Manejo de Errores (Resiliencia): Implementar bloques try-except. Si un archivo HTML está corrupto o no se puede parsear, el script debe registrar un Warning en la consola indicando el nombre del archivo, pero no debe detener el procesamiento del resto del lote.

Dependencias: Incluir un bloque de comentarios al inicio del archivo con las instrucciones de instalación (ej. pip install beautifulsoup4 lxml).
