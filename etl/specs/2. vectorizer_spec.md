# Spec: Document Chunking y Vectorización (RAG Data Pipeline - Paso 2)

## 1. Contexto y Objetivo
Se requiere construir un script en Python (`vectorizer.py`) que tome el corpus de texto limpio generado en el Paso 1, lo fragmente utilizando estrategias de "Chunking" semántico y convierta esos fragmentos en vectores matemáticos (Embeddings). Finalmente, los vectores y sus metadatos deben almacenarse en una base de datos vectorial local (ChromaDB).

## 2. Requerimientos Tecnológicos (Stack)
* **Framework de Orquestación:** `langchain`
* **Modelo de Embeddings:** `HuggingFaceEmbeddings` (Modelo recomendado: `all-MiniLM-L6-v2` para ejecución local eficiente sin costo).
* **Vector Store:** `Chroma` (implementación local persistente).

## 3. Entradas (Input)
* Leer el archivo `corpus_limpio.json` generado en la etapa ETL.
* La estructura esperada es una lista de diccionarios: `[{"id": "...", "titulo": "...", "contenido": "..."}]`

## 4. Procesamiento y Reglas de Negocio
El script debe ejecutar la siguiente secuencia:

1. **Carga de Datos:** Parsear el JSON y convertir cada elemento en un objeto `Document` de LangChain. El `page_content` debe ser el "contenido", y el "titulo" e "id" deben guardarse dentro del diccionario `metadata` del Document.
2. **Chunking (Fragmentación):**
   * Utilizar `RecursiveCharacterTextSplitter` de LangChain.
   * **Chunk Size:** 1000 caracteres.
   * **Chunk Overlap:** 200 caracteres (vital para no perder el contexto semántico entre fragmentos).
   * Aplicar el splitter sobre la lista de documentos cargados.
3. **Generación de Embeddings e Indexación:**
   * Inicializar el modelo de embeddings local con HuggingFace.
   * Inicializar el cliente de `Chroma` configurando un directorio de persistencia local (ej. `./chroma_db`).
   * Inyectar los fragmentos (chunks) y el modelo de embeddings en ChromaDB para que calcule los vectores y los guarde en el disco.

## 5. Salidas (Output)
* Un directorio local llamado `./chroma_db` que contendrá los índices vectoriales en formato SQLite/Parquet (gestionado internamente por Chroma).

## 6. Requerimientos No Funcionales y Calidad de Código
* **Feedback de Consola:** Imprimir métricas clave durante la ejecución (ej. "Total de artículos procesados: X", "Total de chunks generados: Y").
* **Manejo de Dependencias:** Proveer el comando `pip install` necesario como comentario en la cabecera (incluyendo `langchain`, `langchain-community`, `sentence-transformers`, `chromadb`).
* **Idempotencia:** Si el script se corre dos veces, debe limpiar la colección de Chroma antes de insertar, o advertir que la base ya existe para no duplicar vectores.
