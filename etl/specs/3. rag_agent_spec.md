# Spec: RAG Orchestrator con Gemini API (RAG Data Pipeline - Paso 3)

## 1. Contexto y Objetivo
Se requiere construir un script interactivo por línea de comandos (CLI) en Python (`rag_agent.py`) que implemente el patrón RAG. El sistema debe recibir una pregunta, buscar contexto en una base de datos vectorial local (ChromaDB) y utilizar el modelo Gemini de Google (vía LangChain) para generar una respuesta fundamentada exclusivamente en los artículos recuperados.

## 2. Requerimientos Tecnológicos (Stack)
* **Orquestador:** `langchain`
* **Embeddings (Retrieval):** `HuggingFaceEmbeddings` (modelo `all-MiniLM-L6-v2`, ejecución local).
* **Vector Store:** `Chroma` (apuntando al directorio local `./chroma_db`).
* **LLM (Generación):** `ChatGoogleGenerativeAI` de la librería `langchain-google-genai`.
* **Modelo LLM:** `gemini-2.5-flash` (o la versión pro disponible).

## 3. Arquitectura del Flujo (Pipeline)
1. **Inicialización:**
   * Cargar variables de entorno (`dotenv`), asegurando que exista `GEMINI_API_KEY`.
   * Inicializar el modelo de embeddings local.
   * Conectar a la instancia local de ChromaDB.
   * Inicializar el LLM de Gemini con una temperatura baja (ej. `temperature=0.2`) para evitar alucinaciones y forzar respuestas factuales.
2. **Configuración del Retriever:**
   * Configurar Chroma como un retriever que devuelva los top 3 documentos más similares (`search_kwargs={"k": 3}`).
3. **Definición del Prompt (System Message):**
   * Crear un `ChatPromptTemplate` con las siguientes instrucciones:
     "Eres un asistente técnico y estratégico basado en los escritos y conocimientos del autor. 
     Utiliza ÚNICAMENTE los siguientes fragmentos de contexto recuperados para responder a la pregunta. 
     Si no sabes la respuesta o el contexto no contiene la información, di explícitamente 'No tengo información sobre eso en mi base de conocimiento'. No inventes información.
     Contexto: {context}
     Pregunta: {input}"
4. **Construcción de la Cadena (Chain):**
   * Utilizar `create_stuff_documents_chain` para unir el LLM de Gemini y el prompt.
   * Utilizar `create_retrieval_chain` para unir el retriever con la cadena de documentos.

## 4. Interfaz y Salidas (Output)
* El script debe ejecutarse en un bucle `while True` permitiendo ingresar preguntas por consola hasta que se escriba 'salir'.
* **Formato de Respuesta:** Por cada pregunta, imprimir de forma clara:
  1. La respuesta generada por Gemini.
  2. Las "Fuentes utilizadas" (extrayendo el campo `titulo` de los metadatos de los documentos recuperados por Chroma).

## 5. Dependencias
* El script debe incluir en un comentario superior el comando de instalación: `pip install langchain langchain-community langchain-chroma langchain-google-genai sentence-transformers python-dotenv`
