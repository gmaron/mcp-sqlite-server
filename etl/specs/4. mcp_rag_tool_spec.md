# Spec: Integración de RAG como Tool en Servidor FastMCP

## 1. Contexto y Objetivo
Tenemos un servidor Model Context Protocol (MCP) construido con `fastmcp` (v3.x) que actualmente expone herramientas sobre una base de datos SQLite. 
El objetivo es agregar un nuevo sub-servidor o namespace que exponga una herramienta (`@mcp.tool()`) capaz de responder preguntas utilizando una base de datos vectorial local (ChromaDB) y el LLM Gemini, aplicando el patrón RAG.

## 2. Requerimientos Técnicos
* **Framework:** `fastmcp`
* **Base de Datos Vectorial:** `Chroma` apuntando al directorio `./chroma_db` existente.
* **Embeddings:** `HuggingFaceEmbeddings` (modelo `all-MiniLM-L6-v2`).
* **LLM:** `ChatGoogleGenerativeAI` (`gemini-2.5-flash`).

## 3. Especificación de la Herramienta (Tool)
1. **Creación del Sub-servidor:**
   * Crear una instancia de FastMCP para el conocimiento, ej: `knowledge_mcp = FastMCP("KnowledgeBase")`.
   * Montar este sub-servidor en la aplicación principal (`main.py`).
2. **Definición de la Función:**
   * Crear una función asíncrona decorada con `@knowledge_mcp.tool()`.
   * **Nombre propuesto:** `consultar_base_conocimiento_autor`
   * **Parámetro de entrada:** `query: str` (La pregunta o tema a investigar).
   * **Docstring Crítico:** "Utiliza esta herramienta para consultar los artículos, newsletters y el conocimiento personal del autor (Gastón) sobre liderazgo, tecnología, arquitectura cloud, reflexiones personales y management. NO uses tu conocimiento general; usa los resultados de esta herramienta."
3. **Lógica Interna (El Pipeline RAG):**
   * Dentro de la herramienta, inicializar la conexión a Chroma y a Gemini (asegurando el uso de `GEMINI_API_KEY`).
   * Realizar la búsqueda de los 3 documentos más relevantes (`k=3`).
   * Pasar los documentos y la `query` a una cadena de LangChain (`create_stuff_documents_chain` y `create_retrieval_chain`).
   * Formatear el string de retorno devolviendo la respuesta de Gemini seguida de las "Fuentes utilizadas" (títulos de los metadatos).
   * Retornar ese string final. Si ocurre un error, capturarlo y retornar un mensaje de error limpio.

## 4. Consideraciones de Arquitectura Clean
* Mantener la inyección de dependencias o variables de entorno prolijas. 
* Asegurarse de que el middleware de logging existente en el servidor (el que guarda en MongoDB) intercepte y registre las ejecuciones de esta nueva herramienta de RAG para medir su uso y latencia.
